Credential Access Failed Logon Detection T1110 — Brute Force
Wykrywa nieudane próby logowania z agregacją per konto i źródłowe IP. Próg 5 prób w oknie 1h ogranicza szum — dostosuj do środowiska. Wyklucza znane wewnętrzne IP. Oryginalne zapytanie używało okna 1m które w Custom Detection Rule (min. harmonogram 1h) było bezużyteczne — poprawiono na agregację z progiem.
// [Credential Access] Failed logon — aggregated (v2)
DeviceLogonEvents
| where Timestamp > ago(1h)
| where ActionType contains "LogonFailed"
    and RemoteIP !in ('10.x.x.x') // exclude internal scanners
| summarize
    FailCount = count(),
    FirstSeen = min(Timestamp),
    LastSeen  = max(Timestamp),
    arg_max(Timestamp, ReportId, DeviceId)
  by DeviceName, AccountName, RemoteIP
| where FailCount > 5 // adjust threshold per environment
| project FirstSeen, LastSeen, DeviceName,
    AccountName, RemoteIP, FailCount, ReportId, DeviceId
| sort by FailCount desc

Credential Access Failed Login Attempt from Outside Poland T1110 · T1078 — Brute Force / Valid Accounts
Uzupełnienie poprzedniego zapytania — wykrywa nieudane próby logowania spoza Polski. Próba logowania z Rosji, Chin czy Korei jest interesująca nawet jeśli się nie powiodła — może wskazywać na credential stuffing lub aktywne targetowanie konta. Agregacja per konto i kraj eliminuje szum.

Lateral Movement SSH Connection Attempt T1021.004 — Remote Services: SSH
Wykrywa próby nawiązania połączeń SSH zarówno z procesów jak i na poziomie sieci. Wyklucza znane legitne wywołania (diagnostyczne, Azure DevOps). Obsługuje środowiska Windows i Linux.
// [Lateral Movement] SSH connection attempt
let ssh_process_events = DeviceProcessEvents
| where FileName in~ ("ssh", "ssh.exe")
    and (not (ProcessCommandLine has_any (
        "ssh -V", "ssh -Q key", "ssh -Q",
        "/usr/bin/ssh -o SendEnv=GIT_PROTOCOL [email protected]",
        "/usr/bin/ssh [email protected]")))
    and Timestamp between (ago(1h) .. now())
| project Timestamp, DeviceName, InitiatingProcessAccountName,
    InitiatingProcessFileName, ProcessCommandLine, FolderPath,
    DeviceId, ReportId,
    DetectionType = "AttemptedSSHConnection", Source = "Process",
    Platform = iff(FolderPath startswith "/usr", "Linux", "Windows");

let ssh_network_events = DeviceNetworkEvents
| where Protocol == "TCP"
| project Timestamp, DeviceName, InitiatingProcessAccountName,
    InitiatingProcessFileName, InitiatingProcessCommandLine, RemoteIP,
    DeviceId, ReportId,
    DetectionType = "EstablishedSSHConnection", Source = "Network",
    Platform = iff(InitiatingProcessFileName has "ssh", "Windows", "Linux");

ssh_process_events
| union ssh_network_events
| sort by Timestamp desc

Lateral Movement Remote Command Execution (PsExec, PowerShell Remoting) T1021.006 · T1569.002
Wykrywa zdalne wykonanie poleceń przez PsExec lub PowerShell Remoting. Lista wykluczeń obejmuje znane legitne przypadki (Exchange management, AD Sync) — tuning krytyczny dla ograniczenia false positives.
// [Lateral Movement] Remote command execution (PsExec, PowerShell Remoting)
let excludes = pack_array(
    "Enable-DistributionGroup", "Microsoft.Exchange",
    "RemoteExchange.ps1", "Start-ADSyncSyncCycle");
let commands_to_detect = pack_array(
    "-ComputerName", "Invoke-Command",
    "New-PSSession", "Enter-PSSession");
DeviceEvents
| where InitiatingProcessFileName has_any ("psexec.exe", "powershell.exe")
    and (AdditionalFields has_any (commands_to_detect)
        or InitiatingProcessCommandLine has_any (commands_to_detect))
| where not (InitiatingProcessCommandLine has_any (excludes)
    or AdditionalFields has_any (excludes))

Execution · Defense Evasion Suspicious LOLBin Usage T1218 — Signed Binary Proxy Execution
Wykrywa użycie popularnych Living-off-the-Land Binaries. Rozszerzona lista o rundll32.exe i regasm.exe — równie popularne LOLBin często pomijane w regułach. Poprawiono exclusion: każdy plik ma oddzielne wykluczenia żeby uniknąć sytuacji gdzie warunki dla jednego binarnego blokują wykrywanie innego.
// [Execution] Suspicious LOLBin usage (v2 — extended)
DeviceProcessEvents
| where FileName in~ (
    "mshta.exe",
    "regsvr32.exe",
    "certutil.exe",
    "wmic.exe",
    "rundll32.exe",   // added: common LOLBin
    "regasm.exe"       // added: .NET assembly execution
)
// wmic exclusions — monitoring queries
| where not (FileName =~ "wmic.exe" and ProcessCommandLine has_any (
    "cpu get loadpercentage",
    "FreePhysicalMemory"))
// certutil exclusions — legitimate certificate operations
| where not (FileName =~ "certutil.exe" and ProcessCommandLine has_any (
    "-store", "-viewstore", "-addstore"))
// rundll32 exclusions — known system calls
| where not (FileName =~ "rundll32.exe" and ProcessCommandLine has_any (
    "printui.dll", "shell32.dll,Control_RunDLL"))
| project Timestamp, DeviceName, AccountName, FileName,
    ProcessCommandLine, InitiatingProcessFileName,
    InitiatingProcessCommandLine, ReportId, DeviceId

Persistence Suspicious File in Startup Folder T1547.001 — Registry Run Keys / Startup Folder
Wykrywa utworzenie pliku wykonywalnego lub skryptu w folderach Startup. Poprawiono błąd logiczny w oryginalnym zapytaniu — warunek wykluczenia używał and zamiast or, przez co wykluczał tylko sytuację gdy WSZYSTKIE warunki były prawdziwe jednocześnie (praktycznie nigdy). Zmieniono na oddzielne wykluczenia per aplikacja.
// [Persistence] Suspicious file in startup folder (v2 — fixed logic)
let legitApps = pack_array(
    "LegitApp1.exe",   // add known-good startup files
    "LegitApp2.lnk"    // per environment tuning
);
DeviceFileEvents
| where FolderPath has_any (
    "\\Startup",
    "C:\\ProgramData\\Microsoft\\Windows\\Start Menu\\Programs\\Startup")
| where FileName endswith ".exe"
    or FileName endswith ".ps1"
    or FileName endswith ".bat"
    or FileName endswith ".py"
    or FileName endswith ".lnk"
    or FileName endswith ".js"
    or FileName endswith ".vbs"
// fix: separate exclusions with OR (not AND)
| where FileName !in~ (legitApps)
| where FolderPath !contains "F:\\LegitApp"  // add per-app path exclusions
| project Timestamp, DeviceName, FileName, FolderPath,
    InitiatingProcessFileName, InitiatingProcessCommandLine,
    ActionType, ReportId, DeviceId

Reconnaissance Possible Port Scan Detected T1046 — Network Service Discovery
Wykrywa potencjalny port scan na podstawie liczby nieudanych/zablokowanych połączeń z jednego źródłowego IP w oknie 1h. Próg 30 prób — dostosuj do środowiska. Dodano kierunek skanu (Inbound/Outbound) i LocalIP do projektu dla lepszego kontekstu przy triażu alertu. Wyklucza Defender Sense i znane wewnętrzne skanery.
// [Recon] Possible port scan detected (v2)
let base = DeviceNetworkEvents
| where ActionType in (
    "ConnectionAttempt", "ConnectionFailed", "InboundConnectionBlocked")
| where InitiatingProcessParentFileName !contains "Sense"
| where InitiatingProcessCommandLine !contains "PSScript_{"
| where LocalIPType != "Loopback"
| where RemoteIP !in ("10.x.x.x") // exclude internal scanners
| where Timestamp >= ago(1h)
// determine scan direction
| extend ScanDirection = iff(
    ActionType == "InboundConnectionBlocked",
    "Inbound", "Outbound");

base
| summarize
    ScanAttempt  = count(),
    PortsScanned = dcount(LocalPort),
    FirstSeen    = min(Timestamp),
    arg_max(Timestamp, ReportId, LocalIP, ScanDirection)
  by DeviceId, DeviceName, RemoteIP
| where ScanAttempt > 30
| project FirstSeen, Timestamp, DeviceId, DeviceName,
    LocalIP, RemoteIP, ScanDirection,
    ScanAttempt, PortsScanned, ReportId
| sort by ScanAttempt desc

Discovery New Device Discovered (Unonboarded) T1018 — Remote System Discovery
Wykrywa nowe urządzenia w sieci które nie są onboardowane do Defendera. Mechanizm stabilizacji (minSamples) eliminuje "mignięcia" — urządzenie musi być widoczne co najmniej 2 razy w oknie. Wyklucza urządzenia ze wcześniejszych okien czasowych aby ograniczyć szum.
// [Device Discovery] New Device Discovered
let lookback   = 1h;
let minSamples = 2;

let KnownDiscovered =
    DeviceInfo
    | where OnboardingStatus != "Onboarded"
        and MachineGroup == "UnassignedGroup"
    | summarize FirstSeen = min(Timestamp) by DeviceId
    | where FirstSeen < ago(lookback);

let StableInWindow =
    DeviceInfo
    | where Timestamp > ago(lookback)
    | where OnboardingStatus != "Onboarded"
    | summarize Samples = count() by DeviceId
    | where Samples >= minSamples;

DeviceInfo
| where Timestamp > ago(lookback)
| where OnboardingStatus != "Onboarded"
| where DeviceDynamicTags has "Device Discovery"
| where isempty(DeviceManualTags)
| summarize arg_max(Timestamp, ReportId, DeviceName, DeviceId, *) by DeviceId
| join kind=inner StableInWindow on DeviceId
| join kind=leftanti KnownDiscovered on DeviceId
| project ReportId, Timestamp, DeviceId, DeviceName,
    DeviceType, OSPlatform, MachineGroup, DeviceManualTags

Informational Missing Full AV Scan (14+ days) Security Hygiene · Compliance
Wykrywa urządzenia które nie ukończyły pełnego skanu Defendera od ponad 14 dni — obejmuje zarówno anulowane jak i nierozpoczęte skany. Wyklucza urządzenia oznaczone flagą IsExcluded (np. serwery z wyłączonym AV).
// [INFO] Missing Full Scan (14+ days)
DeviceEvents
| extend AdditionalFields = todynamic(AdditionalFields)
| extend ScanType = AdditionalFields.["ScanTypeIndex"]
| where (ActionType contains "AntivirusScanCompleted" and ScanType == "Full")
    or (ActionType contains "AntivirusScanCancelled" and ScanType == "Full")
| join kind=inner (
    DeviceInfo
    | project DeviceName, DeviceId, IsExcluded
) on DeviceId
| summarize arg_max(Timestamp, *) by DeviceName
| where IsExcluded != 1 and Timestamp <= ago(14d)

Informational Device Not Seen for 30 Days Asset Management · Security Hygiene

Informational Mass Email Send (300+ in 5 minutes) T1534 · T1566 — Internal Spearphishing / BEC
Wykrywa konta wysyłające ponad 300 maili w ciągu 5 minut — wskaźnik przejętego konta email lub kampanii BEC. Próg dostosowany do typowego wolumenu wysyłki w środowisku — może wymagać tuningu.
// [INFO] Mass email send — possible BEC / compromised account
EmailEvents
| where Timestamp > ago(5m)
| summarize
    EmailCount = count(),
    ReportId = tostring(min(ReportId)),
    Timestamp = min(Timestamp)
  by SenderFromAddress
| where EmailCount > 300
| project Timestamp, ReportId, SenderFromAddress, EmailCount
