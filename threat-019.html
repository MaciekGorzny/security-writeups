Analiza kodu: Behavioral Bot Detection (CAPTCHA)

Fałszywy widget CAPTCHA na etapie 3 zawierał własnoręcznie napisany kod JavaScript implementujący wielowarstwową detekcję botów. Celem było ukrycie finalnej strony phishingowej przed automatycznymi skanerami URL - narzędziami security, sandboxami i crawlerami, które nie poruszają myszą.

WARSTWA 1Honeypot field
Ukryte pole <input id="honeypot" name="email"> z pozycją left: -9999px - niewidoczne dla użytkownika, ale wykrywalne przez boty z funkcją autouzupełniania. Jeśli pole zawiera jakąkolwiek wartość → bot detected, CAPTCHA reset bez przekierowania.
WARSTWA 2Time gate (800 ms)
Kliknięcie w ciągu pierwszych 800 ms od załadowania strony → bot detected. Automaty często klikają natychmiast po załadowaniu DOM; prawdziwy użytkownik potrzebuje chwili na zlokalizowanie widgetu.
WARSTWA 3Mouse movement tracking + wariancja prędkości
Skrypt rejestruje do 100 ostatnich punktów ruchu myszy (mousemove), oblicza prędkość między kolejnymi punktami i ich wariancję. Warunek odrzucenia: jeśli avg > 0.2 px/ms AND variance < 0.0005 → ruch jest zbyt równomierny → bot detected. Prawdziwy ruch myszy charakteryzuje się naturalną nieregularnością; automaty często symulują jednostajne przesuwanie.
BYPASSMobile auto-pass
Jeśli navigator.userAgent zawiera Mobi|Android|iPhone → isHuman() zwraca true bez dalszej weryfikacji. Na urządzeniach mobilnych dane ruchu myszy są niedostępne, więc sprawdzanie wariancji jest pomijane.
UX DECEPTIONLosowe opóźnienie + animacja ładowania
Po kliknięciu: stan loading (animacja spinner) przez losowe 900 + Math.random() * 500 ms (0.9–1.4 s), następnie checkmark i kolejne 600 ms przed przekierowaniem. Symuluje prawdziwą weryfikację i utrudnia identyfikację jako fałszywy widget.
Efekt detekcyjny: Większość automatycznych skanerów URL (sandboxe, crawlery, reputacyjne systemy threat intelligence) nie generuje naturalnego ruchu myszy. Skrypt skutecznie ukrywa finalną stronę credential harvestingu przed tymi narzędziami. Komentarz w kodzie // ← change this przy stałej REDIRECT wskazuje na gotowy template wielokrotnego użytku - kampania prawdopodobnie nie jest jednorazowa.
// Fragment kodu: funkcja isHuman() - wielowarstwowa detekcja botów function isHuman() { // WARSTWA 1: honeypot if (honeypot.value !== '') return false; // WARSTWA 2: time gate if (Date.now() - loadTime < 800) return false; // BYPASS: mobile UA if (/Mobi|Android|iPhone/i.test(navigator.userAgent)) return true; if (mousePoints.length < 3) return false; // WARSTWA 3: analiza wariancji prędkości myszy if (mousePoints.length >= 6) { const speeds = mousePoints.slice(-10).map(p => Math.sqrt(p.dx * p.dx + p.dy * p.dy) / Math.max(p.dt, 1)); const avg = speeds.reduce((a, b) => a + b, 0) / speeds.length; const variance = speeds.reduce((s, v) => s + Math.pow(v - avg, 2), 0) / speeds.length; if (avg > 0.2 && variance < 0.0005) return false; // ruch zbyt jednostajny = bot } return true; }

Rekomendacje detekcyjne
KQL - Sentinel / Defender XDR: Wykrywanie emaili z domeny .jimdofree.com w URL
EmailUrlInfo | where Url has "jimdofree.com" | join kind=inner EmailEvents on NetworkMessageId
KQL - Wykrywanie domen creaters.info w ruchu sieciowym
DeviceNetworkEvents | where RemoteUrl has "creaters.info" or RemoteUrl has "encoded-document"
Reguła behawioralna: Email z Importance: high + pusty temat lub jednowyrazowy (FYI, URGENT, CHECK) + zewnętrzny nadawca + obraz z zewnętrznym URL w treści → priorytetowa analiza nagłówków.

Pattern behavioral CAPTCHA: Strony z JS zawierającym jednocześnie: honeypot field (left: -9999px), mousemove tracking + variance calculation, window.location.href = REDIRECT po setTimeout - wzorzec evasion do dodania do reguł YARA-L / Sigma dla proxy/EDR.
Uwaga na SPF softfail ≠ spoofing: Email zdał SPF na warstwie Mimecast (wyszedł z prawdziwego M365), ale dostał softfail od strony Exchange Online odbiorcy z powodu relay przez Mimecast. Automatyczne reguły odrzucające wyłącznie na podstawie SPF softfail mogłyby dawać false positives w podobnych scenariuszach. Kluczowe jest zestawienie SPF + DKIM + DMARC + analiza pełnej ścieżki dostarczenia.
