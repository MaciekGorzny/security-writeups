🔍
Kluczowe ustalenie: zorganizowana kampania affiliate. Pięć emaili z różnych domen nadawcy w ciągu 10 dni, prowadzących do tych samych domen końcowych. Kod źródłowy pośredniej strony ujawnił pełną infrastrukturę affiliate trackingu — parametry sub1-sub5, encoded_value i dedykowany tracker konwersji trk-brevinate.com. To nie jest jednorazowy phishing — to jest skalowalna kampania gdzie różni "affiliates" rozsiewają maile i zarabiają prowizję od każdej ofiary kupującej "Total Drive".
⚠️
Technika: Scareware + social engineering finansowy. Kampania używa dwóch różnych lure'ów prowadzących do tej samej infrastruktury: (1) iCloud storage limit — "pliki usunięte za 24h", (2) scareware "TRWA PRZEJĘCIE KONTA · BLOKADA SYSTEMU". Cel identyczny w obu przypadkach — skłonić ofiarę do zakupu fałszywego "antywirusa" lub "cloud storage" za 19€ podając dane karty kredytowej.
💡
Klasyfikacja: Scareware / Tech Support Scam / Crapware. Produkty "Total Drive" i "Total Adblock" są prawdopodobnie realnie istniejącymi, ale bezużytecznymi lub złośliwymi aplikacjami sprzedawanymi przez agresywną sieć affiliate. Celem jest zarówno kradzież danych karty jak i wyłudzenie opłaty subskrypcyjnej (19€ → 99€/rok auto-odnowienie).
Executive Summary

W okresie 16–26 marca 2026 na prywatną skrzynkę pocztową trafiło 5 emaili phishingowych z różnych domen nadawcy, wszystkie prowadzące do tej samej infrastruktury kampanii. Maile używały dwóch lure'ów: fałszywe powiadomienie iCloud o przepełnieniu miejsca oraz scareware "TRWA PRZEJĘCIE KONTA · BLOKADA SYSTEMU".

Analiza łańcucha ataku ujawniła 3-etapową infrastrukturę: pośrednia strona na impawnunsung[.]motorcycles (fake iCloud/Cloud storage) → affiliate redirect przez twointj.com z pełnym parametryzowanym trackingiem → finalna strona harvestingu karty na secure.totaldrive[.]com lub secure.totaladblock[.]com. Inspekcja kodu źródłowego pośredniej strony ujawniła pełną architekturę systemu affiliate — parametry sub1-sub5, encoded_value i dedykowany tracker konwersji trk-brevinate[.]com.

Kampania jest prowadzona przez sieć affiliate gdzie różni operatorzy (różne domeny nadawcy emaili) kierują ruch do wspólnej infrastruktury sprzedażowej i zarabiają prowizję od każdej "sprzedaży". Ofiara jest dwukrotnie poszkodowana: płaci 19€ za bezużyteczny produkt i oddaje dane karty do dalszego nadużycia.

Każdy email wysłany przez innego "affiliate" zawiera unikalny zestaw parametrów trackingowych który identyfikuje źródło ruchu i pozwala na rozliczenie prowizji. Ruch przechodzi przez twointj.com (redirect broker) i trk-brevinate.com (konwersja tracker) zanim dotrze do finalnej strony zakupu.

// Kod źródłowy impawnunsung.motorcycles — affiliate tracking JavaScript
// Ujawniony przez inspekcję view-source w przeglądarce

// Redirect broker z zakodowanymi parametrami affiliate
var redirect = "https://www.twointj.com/cmp/2SFN28H/42P5QX7/";

// Parametry sub1-sub5 = identyfikatory affiliate, kampanii, źródła
// encoded_value = zakodowany identyfikator ofiary/sesji
document.getElementById('button1').addEventListener("click", function(){
    var urlParams     = new URLSearchParams(window.location.search);
    var encodedValue  = urlParams.get('encoded_value');  // victim session ID
    var source_id     = urlParams.get('source_id');      // traffic source
    var domain        = urlParams.get('domain');
    var sub1          = urlParams.get('sub1');           // affiliate ID
    var sub2          = urlParams.get('sub2');           // campaign ID
    var sub3          = urlParams.get('sub3');
    var sub4          = urlParams.get('sub4');
    var sub5          = urlParams.get('sub5');

    // Redirect do merchantu z pełnym trackingiem konwersji
    window.location.href = "https://" + domain +
        "/cmp/" + encodedValue +
        "/73H5RHN/?source_id=" + source_id +
        "&sub1=" + sub1 + "&sub2=" + sub2 +
        "&sub3=" + sub3 + "&sub4=" + sub4 +
        "&sub5=" + sub5;
});

// Konwersja tracker — rejestruje każde kliknięcie
script.src = "https://trk-brevinate.com/scripts/push/script/o7g9x7qg36?" +
    "url=" + encodeURI(self.location.hostname) +
    "&alturl=" + encodeURI(self.location.pathname);
script.onload = function() { push_init(); push_subscribe(); };
Co ten kod ujawnia o infrastrukturze atakujących:
Parametr domain w URL pozwala operatorowi kampanii dynamicznie zmieniać domenę końcową bez modyfikacji pośredniej strony — stąd ofiara trafia raz na totaldrive.com, raz na totaladblock.com. To jest standardowa technika w sieciach affiliate pozwalająca na rotację ofert. push_subscribe() sugeruje dodatkowo że strona próbuje zapisać się na push notifications — kolejny wektor do dalszego spamowania ofiary.
Analiza techniczna — etap po etapie
STAGE 1Email — dwa lure'y, pięć nadawców
Lure A — iCloud storage: Mail z logo iCloud informujący o przekroczeniu limitu 15 GB (14,9/15 GB), odrzuconej płatności za iCloud+ 2TB i grożący trwałym usunięciem plików za 24h. Przycisk "Odnów dostęp". Wysłany z czterech różnych domen nadawcy w ciągu tygodnia.

Lure B — Scareware: Mail z nazwą nadawcy iCL0UD (zero zamiast litery O — typosquatting w nazwie wyświetlanej) z komunikatem "TRWA PRZEJĘCIE KONTA · BLOKADA SYSTEMU · Automatyczny protokół czyszczenia został aktywowany". ID zdarzenia: STG-PURGE-9912X, poziom zagrożenia: KRYTYCZNA UTRATA, czas odzyskiwania: <12 godzin. Przycisk "ZATRZYMAJ USUWANIE DANYCH".

Oba lure'y celują w ten sam mechanizm psychologiczny — natychmiastowa panika i strach przed utratą danych. Różne opakowanie, identyczna infrastruktura docelowa.
STAGE 2impawnunsung[.]motorcycles — fake Cloud storage landing page
Strona imituje panel zarządzania miejscem w chmurze — kolorowy pasek pojemności, kategorie (Zdjęcia i filmy, Pliki, Kontakty, Poczta, Kopie zapasowe), "Pozostało tylko 0,3 GB". Przycisk "Ulepsz pamięć".

DevTools ujawniają zasoby z: MetaMask, Yoroi (crypto wallet), trk-brevinate.com (affiliate tracker), use.fontawesome.com. Obecność MetaMask i Yoroi w zasobach jest nieoczekiwana i może wskazywać że strona obsługuje również wariant ataku targetujący portfele kryptowalutowe.

meta name="robots" content="noindex, nofollow" — strona celowo ukrywa się przed indeksowaniem przez wyszukiwarki, co utrudnia wykrycie przez automatyczne skanery bezpieczeństwa.

UI — precyzyjna imitacja Apple iCloud Storage: Pasek pojemności jest podzielony na 5 nazwanych segmentów: seg-media (Zdjęcia i filmy), seg-docs (Pliki), seg-contacts (Kontakty), seg-mail (Poczta), seg-device (Kopie zapasowe urządzenia) — identyczna struktura jak w oryginalnym panelu iCloud. Każda kategoria ma animowany spinner "ładowania" (fa-spinner fa-spin) zastępowany statusem "Done" przez JavaScript, co symuluje faktyczne skanowanie konta ofiary i zwiększa wiarygodność.

Wielojęzyczność — zasięg globalny: Cały widoczny tekst strony jest ładowany dynamicznie przez system data-i18n z zewnętrznego ./js/app.js. Oznacza to że jedna infrastruktura obsługuje wiele wersji językowych — kampania nie jest targetowana tylko w jeden kraj/język, lecz jest skalowalna globalnie z jednym backendem.

Push notification persistence — dodatkowy wektor: Kod źródłowy ujawnia że trk-brevinate.com to nie tylko tracker konwersji — strona aktywnie próbuje zapisać ofiarę do push notifications przez przeglądarkę: 
// trk-brevinate.com — push subscription loader (linie 178–187 kodu źródłowego)
var script = document.createElement("script");
script.type = "text/javascript";
script.src = "https://trk-brevinate.com/scripts/push/script/o7g9x7qg36" +
    "?url=" + encodeURI(self.location.hostname) +
    "&alturl=" + encodeURI(self.location.pathname);
script.onload = function() {
    push_init();      // inicjalizuje push notification API
    push_subscribe(); // wyświetla prompt przeglądarki "Zezwól na powiadomienia"
};
document.getElementsByTagName("head")[0].appendChild(script);
⚠ Implikacja — persistence po opuszczeniu strony:
Jeśli ofiara kliknie "Zezwól" na prompt przeglądarki, atakujący uzyskują kanał do dalszego spamowania push notyfikacjami — niezależnie od tego czy ofiara kiedykolwiek wróci na stronę. Przeglądarkowe push notifications wyglądają jak systemowe alerty systemowe i mogą być używane do kolejnych lure'ów (np. "Twoje konto zostało zablokowane — kliknij tutaj"). To jest technika persistence bez plików, bez instalacji — czysto przez Web Push API.
STAGE 2bAnti-back-button — zatrzymanie ofiary w lejku
Kod <head> strony zawiera mechanizm blokady przycisku Wstecz oparty na manipulacji historią przeglądarki: 
// Linie 18–32 kodu źródłowego — anti-back-button via History API
var redirect = "https://www.twointj.com/cmp/2SFN28H/42P5QX7/";
var currentUrl = location.origin + location.pathname + location.search;

// Przy wejściu na stronę: dodaj #!/hst do historii (pułapka)
if (location.hash !== "#!/hst") {
    history.replaceState(null, document.title, currentUrl + "#!/hst");
    history.pushState(null, document.title, currentUrl); // aktualny URL na wierzchu
}

// Gdy ofiara kliknie Wstecz (popstate) → trafia na #!/hst → redirect do twointj
window.addEventListener("popstate", function() {
    if (location.hash === "#!/hst") {
        setTimeout(function() { window.location.replace(redirect); }, 0);
    }
}, false);
Jak to działa: Przy wejściu na stronę JavaScript wstrzykuje do historii dodatkowy wpis z hashem #!/hst. Gdy ofiara klika "Wstecz" żeby uciec, przeglądarka cofa się do tego wpisu, co triggeruje popstate event, który natychmiast przekierowuje do twointj.com — czyli ofiara zamiast uciec, trafia głębiej do lejka affiliate. Jedyna ucieczka to zamknięcie zakładki lub wpisanie URL ręcznie.

Nota o campaign ID: Hardcoded redirect (42P5QX7) różni się od identyfikatora w URL tracking (73H5RHN) — wskazuje na dwa różne "sloty" affiliate lub warianty oferty w tej samej sieci. Strona może serwować różne finalne produkty w zależności od tego którą ścieżką trafi ofiara.
STAGE 3freesmartdevice[.]autos — scareware overlay
Po kliknięciu "Ulepsz pamięć" na tle ciemnego ekranu pojawia się pop-up: "Your Device Has Been Compromised!" z komunikatem że malware pobrało aplikacje, ukryło je i włączyło auto-billing bez zgody użytkownika. Następnie kolejny pop-up: "Enabled Auto-Renewable Subscription Is Detected — Is That You?" z fałszywymi danymi:

· Transaction ID: 298606940829486298
· Subscription: File Downloader (Weekly Plan)
· Amount: $299.00, then $29.99 weekly
· Subscription Started From: 2026-04-02 20:17

To jest klasyczny scareware pattern: najpierw wzbudzenie paniki ("twoje urządzenie jest zainfekowane"), następnie konkretna kwota ("już Ci pobierają pieniądze") żeby skłonić ofiarę do kliknięcia "Fix Now" i zakupu fałszywego antywirusa.
STAGE 4secure.totaldrive[.]com / secure.totaladblock[.]com — card harvesting
Finalna strona to formularz zakupu "Total Drive" lub "Total Adblock Plus" — w zależności od wariantu kampanii. Struktura identyczna w obu przypadkach:

· "Super okazja! Oszczędź 80% Off" — fake discount pressure
· Stawka podstawowa: 99€, dziś tylko: 19€
· Drobny druk: "automatycznie odnawia się co roku za 99€"
· Formularz: imię i nazwisko, numer karty, data ważności MM/YY, kod zabezpieczający, adres, kod pocztowy, email
· Trustpilot ★★★★★ (fałszywe lub manipulowane oceny)

DevTools ujawniają integracje z: cdn.paymentauth.com, cdn.prod.pci-bridge.com, pay.google.com, www.paypal.com — prawdziwe bramki płatnicze co sugeruje że transakcja faktycznie jest realizowana (ofiara płaci 19€ i dostaje bezużyteczny produkt), a dane karty są dodatkowo harvested przez pci-bridge.com.

Działania i rekomendacje
⚠ Red flags tej kampanii:
· Apple nigdy nie wysyła powiadomień o iCloud z domen innych niż @apple.com lub @icloud.com
· "iCL0UD" z zerem zamiast O w nazwie nadawcy — typosquatting w display name jest niewidoczny jeśli nie sprawdzić adresu email
· Żaden legitny komunikat bezpieczeństwa nie zawiera komunikatu "TRWA PRZEJĘCIE KONTA" z odliczaniem czasu — to jest klasyczny scareware pattern
· Strona z Trustpilot ★★★★★ + "Gwarancja zwrotu pieniędzy" + agresywny rabat 80% = silny sygnał scamu
· Formularz kart kredytowej na stronie innej niż apple.com = bezwzględny red flag
💡 Threat Intelligence — jak działa ten model biznesowy:
Kampania opiera się na modelu Cost Per Acquisition (CPA) typowym dla legalnych sieci affiliate — z tą różnicą że produkt jest scamem. Każdy "affiliate" (operator domeny nadawcy) rejestruje się w sieci, dostaje unikalny zestaw sub-parametrów i rozsyła spam. Za każdą "sprzedaż" (ofiara która zapłaci 19€) dostaje prowizję. Sieć (twointj.com) pobiera swoją marżę. Merchant (totaldrive.com) realizuje transakcję i zbiera dane karty.

Model ten jest trudny do zwalczania bo każdy element łańcucha jest łatwo wymienialny — zmiana domeny nadawcy, zmiana pośredniej domeny, rotacja ofert. Jedynym stałym elementem jest sieć affiliate i tracker konwersji.
